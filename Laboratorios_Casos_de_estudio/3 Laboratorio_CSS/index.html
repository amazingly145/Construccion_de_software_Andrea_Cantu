Preguntas Laboratorio 1 Introduccion a HTML

1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

Respuesta: El internet es una inmensa red de computadoras conectadas entre si, mientras que la World Wide Web (WWW) es una enorme colección de páginas que se sienta sobre esa red de computadoras

2. ¿Cuáles son las partes de un URL?

Respuesta: Una URL (Localizador Universal de Recursos) es un tipo de URI (Identificador Universal de Recuros) específico, un URL se ubica normalmente en un recurso ya existente de internet, el mismo se utiliza cuando un cliente web hace una petición al servidor para obtener un recurso. El URL en el inicio tiene el protocolo, seguido por el dominio, el path y el parametro.

3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Respuesta: Los métodos HTTP, son el formato de comunicación entre el cliente y el servidor web.
Los metodos mas comunes son GET y POST:

  • GET: realiza una petición a un recurso en específico. Los datos no son modificados ni los estados de los recursos
  • POST: creamos un nuevo recurso, necesita de un cuerpo en donde defines la identidad de la data creada, puede enviar datos por medio del cuerpo para crear el recurso.
  • PUT: usas este metodo para actualizar un recurso existente, utilizando el contenido del mismo, es una forma de "editar" algo. Si no hay un recurso que sea igual al que se pide, se crea uno nuevo.
  • HEAD: usas este metodo igual que GET, pero solo regresa HEAD, no debe de tener el cuerpo, usas el metodo par verificar si el recurso esta presente a la hora de usar GET.
  • PATCH: Usas el metodo para aplicar parcialmente las modificaciones del recurso, con patch solo necesitas pasar la data que quieres modificar
  • DELETE: Usas el metodo para eliminar el recurso especificado.

4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Respuesta: Se debe de utilizar el metodo de POST, porque estas creando una nueva respuesta con el usuario y contraseña. Pero para obtener esta respuesta, debes de usar el metodo GET para visualizar el usuario y contraseña.

5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Respuesta: Se utiliza el método de GET porque quieres tener acceso a un recurso, en este caso lo que se tiene dentro de la URL.

6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Respuesta: Cuando un servidor web regresa el codigo 200, significa que la solicitud se hizo exitosamente. Lo cual signiifica que para los metodos:

  • GET: los recursos han sido obtenidos y transmitidos en el mensaje del cuerpo.
  • HEAD: Los headers se han incluido en la respuesta sin un mensaje del cuerpo.
  • PUT o POST: Los recursos se han transmitid en el mensaje del cuerpo.

En resumen: No significa que hay un error, por lo contrario que se hizo exitosamente lo pedido.

7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Respuesta: El error 404 o "Not Found", es cuando el servidor no puede encontrar el recurso solicitado, en el buscador significa que el URL no es reconocido.
En API tambien puede significar que el endpoint es valido pero el recurso en sí no existe.

Sí es respondabilidad del desarollador porque tiene que ver porque el servidor no puede encontrar el recurso y arreglarlo, para que se pueda visualizar.

8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Respuesta: El error 500 o "Internal Server Error", este es un error genérico, indicando que el servidor no puede encontrar un codigo de error mas apropiado Tambien pueden significar fallos tecnicos dentro del codigo y sí es responsabilidad del desarrollador corregir el error, para brindarle una mejor experiencia al usuario y al cliente.
Además que es la unica manera que se puede visualizar la página web

9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Respuesta: Cuando una etiqueta se dice que esta despreciado o desaprobado, significa que su uso no se recomienda porque esta desactualizado o se generaron nuevas alternativas. Algunos elementos de html 4 que ya no estan en html 5 son: font, applet, center, big, basefont, font, menu, dir, acronym, etc.

10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Respuesta:

  • HTML 4: era un estándar centrado en la estructura básica de un documento y en la separación de contenidos. Se necesitaba el uso extendido de divs y clases para represetar la estructura de header, nav, article, section y footer. Esto complicaba la interpretación por motores de busqueda
  • HTML 5: representa una evolución al desarrollo web, incorpora nuevas etiquetas semánticas, APIs nativos del navegador y soporte multimedia que facilita técnicas modernas de seguridad. Tiene etiquetas de audio y video sin usar plugins adicionales. También, implementa etiquetas de sintaxis como: header, nav, article, section y footer

11. ¿Qué componentes de estructura y estilo tiene una tabla?

Respuesta: Una tabla es un conjunto de datos estructurados en filas y columnas. Esto permite buscar con falcilidad y rapidez los elementos. Para hacer una tabla es neccesario utilizar los metodos tr que hace la tabla y dentro del mismo usar td. Para darle estilo a la misma es necesario usar hojas de estilo.

12. ¿Cuáles son los principales controles de una forma HTML5?

Respuesta: Los principales controles de una forma HTML5 se basa en la etiqueta input de HTML4, pero con nuevos tipos especifico como email, numerico, url, sliders, fecha, color, busqueda y telefono.

  • ¿Cuál es el ciclo de vida de los sistemas de información?

    • Respuesta: Ciclo de vida de un sistema de información, es un sistema manual, que engloba a personas, máquinas o métodos organizados. para recopilar, procesar, transmitir datos que represente información. Vida del proyecto. El mismo contiene las siguiente etapas:

    1. Planificación: Realizar una serie de tareas o actividades que van a influir en el éxito del proyecto
    2. Análisis: buscar que es exactamente lo que tiene que hacer el sistema y se llega a una compresión adecuada de los requerimientos del sistema.
    3. Diseño: Se estudian diferentes alternativas para solucionar la problemática y se decide la estructura en general que tendrá el sistema.
    4. Implementación: Se seleccionan las herramientas necesarias para su implementacion y un lenguaje apropiado para la misma.
    5. Pruebas: Su objetivo principal es detectar los errores que se hayan tenido en etapas anteriores y después corregirlos.
    6. Instalación o despliegue:Se debe de planificar los sistemas que va a funcionar.
    7. Uso y mantenimiento: Ya después de tener el producto, se le debe de dar mantenimiento al programa para que continue con su correcto funcionamiento.

Preguntas Laboratorio 2 Hojas de estilo

1. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

Respuesta: El uso de !importante es muy importante, porque es la regla de una hoja de estilo que nunca va cambiar, sin importar que se cambie en el id o clase, mi recomendacion para usarla seria que pensemos en las reglas que queremos que se respeten y nunca cambien, el cual le colocaremos !important

2. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Respuesta: Cuando se pone una imagen de fonde, debemos de tener cuidado en que la imagen deje que se pueda leer el texto correctamente, tambien puede hacer que la navegacion sea lenta por lo que deberia de quitarse si es demasiado lenta

3. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

Respuesta: Cada unidad tiene su distinctivo o una razon por la que deberia de usarse.

  • %: es un valor que se establece en relacion a otro, si pones un font size en porcentaje, entonces sera el porcentaje del elemento padre. Se usa mayormente para: layouts, anchos de contenedores y elementos que debene adaptarse a tamanos de la pantalla (dinamicos).
  • px: Son los pixeles en la pantalla, unidades absolutas, para elementos que no deben de cambiar de tamano como: bordes, margenes, paddings. Esto proporciona controle exacto.
  • pt: Son unidades absolutas de impresion, evitar de usar en diseños web, estan diseñados mejor para documentos físicos y no se ajustan muy bien a la pantalla, reduciendo su accesibilidad.

4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Respuesta: Debido a que minimiza el archivo, hace que la carga del sitio sea mas rapido y se tarde menos tiempo en cargar. Elimina espacios, comentarios o caracteres innecesarios
