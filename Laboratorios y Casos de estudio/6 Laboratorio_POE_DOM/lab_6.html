Preguntas Laboratorio 6

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Ayuda a garantizar que los datos que se envían coinciden con los requisitos establecidos en los diferentes controles de formularios, si se detectan datos no válidos, se podrá corregir rápidamente. Si el servidor lo recibe y lo rechaza, se crea un rechazo entre en la comunicación entre el servidor y el cliente.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Puedes editar el HTML, usando atributos como novadilate en la etiqueta form o eliminar el atributo rquiered en la label form o puedes cambiar el tipo de input a solo texto

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Porque evita errores, es decir, si alguien escribe una ciudad mal o edad, etc. Entonces el formulario lo detectara. Tambien de esta manera evitamos tener datos nulos y los podemos tener completos, ayuda a detectar errores como de formato rápidamente, sin esperar la respuesta del servidor lo que hace la web mucho más rápida, evita enviar datos invalidos al tiempo, lo que ahorra mucho tiempo
